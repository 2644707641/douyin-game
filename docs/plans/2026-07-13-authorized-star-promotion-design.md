# 指定账号段位升星测试功能设计

## 目标

在账号面板中为精确账号名 `xuwang` 提供“段位 +1 星”测试按钮。其他账号及未登录用户不可见、不可调用。操作成功后存档立即上传并刷新游戏。

## 架构

- 只修改 `web-platform-patch.js`，不侵入压缩的 `bundle.js`。
- 授权来源为已经登录并由账号 RPC 返回的 `state.session.user.account`。
- UI 在现有账号面板中增加独立测试区，由 `updatePanelState()` 统一控制可见性和禁用状态。
- 操作处理器再次校验账号，避免仅依赖 DOM 隐藏。

## 数据流

1. 严格校验当前登录账号等于 `xuwang`。
2. 从 `Laya.LocalStorage` 读取既有 `playerData`。
3. 校验 JSON、对象结构及 `_curStar` 数值，拒绝缺失或损坏的存档。
4. 将 `_curStar` 增加 1，并更新 `_saveTime`。
5. 写回 `Laya.LocalStorage`，触发云存档覆盖层。
6. 调用 `flushNow({ force: true, throwOnError: true })` 确认上传。
7. 同步账号资料中的 `bestStar`。
8. 刷新到当前存档 URL，让 bundle 重新读取玩家数据。

## 边界与错误处理

- 非 `xuwang`：按钮隐藏且操作函数拒绝执行。
- 存档缺失、JSON 损坏或 `_curStar` 非有限数字：报错，不覆盖原存档。
- 星级上限设为 `999999`；达到上限时报错。
- 云上传失败：恢复操作前的原始 `playerData`，再次强制写回，且不刷新页面。
- 不修改 `_lastStar`，避免干扰每日段位奖励。

## 测试策略

- 抽取并执行生产函数源码，验证授权判断和星级更新逻辑。
- 验证非授权、合法升星、损坏存档、上限和上传失败回滚。
- 运行现有跨设备同步回归测试、JavaScript 语法检查和浏览器页面验证。

## 安全说明

这是客户端测试入口隔离，不是不可绕过的服务端权限。真实强授权需要新增 Supabase RPC 和数据库权限规则，不在本次已确认范围内。
