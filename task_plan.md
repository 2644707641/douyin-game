## 关键问题
1. 云存档是否在登录后正确绑定到稳定的账号 ID？
2. 第二设备是否实际读取了云端数据，还是本地默认数据覆盖了云端？
3. 写入是否被鉴权、时序、字段格式或版本冲突静默拒绝？
4. 游戏“一颗星”对应 `playerData` 中哪个字段，是否存在现成 setter？
5. 提升后是否需要刷新页面才能让游戏内存与存储一致？
6. 最早版本中商人是按哪个段位 ID 解锁，`openProps` 是否为永久标记？

## 决策记录

| 决策 | 原因 |
|------|------|
| 将现有未提交改动视为用户工作 | 防止覆盖未知来源的修改 |
| 先调查根因再修改 | 遵循系统化调试，避免症状补丁 |
| 不创建真实生产测试账号 | 注册/登录会写生产 Supabase，按危险操作规则不得未经确认执行 |
